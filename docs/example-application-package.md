# 示例申请包预览

用户选择某个机会后，自动化会在 `generated/applications/` 下生成真实申请包。测试/模拟运行会写到 `generated/test-runs/` 或 `generated/mock-runs/`。

```text
generated/applications/12-example-residency/
  internal-notes/
    open-call-analysis.md
    portfolio-research.md
    quality-report.json
    internal-issues.md
  user-review/
    机会与风险-中文审核.md
    最终提交前检查清单-中文.md
    英文正式材料中文说明.md
    application-package.docx
    application-package.pdf
  external-submission/
    application-answers-en.md
    bio-en.md
    statement-en.md
    cv.md
    email-en.md
    portfolio-text.md
    portfolio.html
    portfolio.pdf
    file-checklist.md
    images/
      work-01.jpg
      work-02.jpg
  opportunity.json
  package-manifest.json
```

## 文件边界

- `internal-notes/` 可以写匹配逻辑、风险、缺失信息、作品选择理由、AI 判断过程和质量问题。
- `user-review/` 默认中文，帮助用户判断机会、风险、材料、AI 做了哪些修改，以及最终提交包是否可提交。
- `external-submission/` 只放机构应该看到的正式内容，不出现 `draft`、`generated by AI`、`internal note`、`placeholder`、`N/A`、`TBD`、`missing`、`unknown` 等词。

## Manifest 示例

```json
{
  "manifestVersion": 2,
  "status": "package_ready_for_final_review",
  "runMode": "real",
  "boundaryModel": "internal_notes/user_review/external_submission",
  "userReviewNodes": ["choose_opportunities", "approve_final_submission_package"],
  "requiresUserFinalApproval": true,
  "directories": {
    "internalNotes": "internal-notes",
    "userReview": "user-review",
    "externalSubmission": "external-submission"
  }
}
```

`external-submission/images/` 只会复制项目允许目录中的图片：`artist-assets/works/`、`artist-assets/source-materials/` 和 `artist-assets/inbox/`。其他系统路径会被跳过。
