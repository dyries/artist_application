# Submission Boundary Rules

Generated application folders must contain three separated areas:

- `internal-notes/`: matching logic, risks, missing information, work-selection rationale, AI judgment, open call analysis, quality issues.
- `user-review/`: Chinese-first opportunity analysis, recommendation, risk explanation, material summary, CV/portfolio changes, final checklist.
- `external-submission/`: formal institution-facing files only.

External files must not contain internal workflow language, AI traces, placeholder terms, or negative missing-information terms such as `draft`, `final candidate`, `ready-to-copy`, `selected for this open call`, `for this residency`, `FILE NOT FOUND`, `generated by AI`, `internal note`, `review version`, `application package`, `to be edited`, `placeholder`, `No website`, `No Instagram`, `None`, `N/A`, `to be confirmed`, `TBD`, `missing`, or `unknown`.

When information is missing, omit it from external files and record it in `internal-notes/internal-issues.md`.
